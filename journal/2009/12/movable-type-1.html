写真で振り返る「Movable Type エンジニアの夕べ」

2009年12月10日 Movable Type 柳谷 真志

みなさん、こんにちは、まーしーです。
寒くなってきて朝起きるのがどうにもこうにもつらいですね。布団から出るのは己との戦いの毎日です。

「Movable Type エンジニアの夕べ」に参加してきました

先日、渋谷でスカイアークシステムさん主催で行われました「Movable Type エンジニアの夕べ」に参加してきました。

「Movable Type エンジニアの夕べ」へのご招待 | ブログからのお知らせ | MTエンジニアブログ | スカイアークシステム
http://www.skyarc.co.jp/engineerblog/entry/movable_type.html

MTエンジニアとは言えるほどではないのですがプラグイン等をがんがん開発されてる方達のお話しが聞いてみたいと思い参加しました。
内容はLightning Talks 形式で色々な話題が取り上げられてました。
サブタイトルが「Lightning Talks ? ソース(source/sauce)だけでご飯3杯！」 というのも熱いですね！そのLightning Talksの内容を振り返ってみましょう。  

テンプレート作成をもっと便利にするMT5プラグインの紹介

スカイアークシステムの小林さんのライトニングトークはMTでよく使うモノがMTタグとして既に用意されてたりするといいよね！というお話し
パンくずリストも作ろうと思えばつくれるけど、MTタグ１つで実装できちゃうといいよねぇ、といったお話し。

そういう要望があるとぴろりさん（上西さん）がサクっと作ってしまうらしいです。スバラシイ！

Google Wave For Movable Type

続いてのお話しはToI企画の天野さん。
MTとGoogle Waveを連携してしまうプラグインのお話しです。


既に資料が公開されています。  

MT > Google Wave For Movable Type > Google Wave for Movable Type の紹介 - ToI
http://tec.toi-planning.net/mt/wave/demo/  

どのLightning Talkも面白く、内容もすごく濃いものだったんですが、会場が一番沸いたのはこのプラグインかもしれません。
Waveが開発中なので、実際に使えるようになるのはまだまだ先なのかもしれませんが、１つのblipから１エントリーとして対談形式のエントリーを生成したり、複数のエントリーに分けて投稿などが出来る様です。

Wave上でこんなかんじでかくと

このように反映されるようです。すごすぎる！


中身としてはXML-RPCの拡張とかおっしゃってました。
MTの深いところの話もわからないといけないですし、Waveのカスタマイズ方法などもわからないと話にならないようです。
Waveは招待されたけれどもどう使ったものかな？といっているようでは遅いですね（苦笑

プラグインの下位互換性を最大化する

続いてはスカイアークシステムのぴろりさん（上西さん）のお話しで、MT5向けにつくったプラグインがMT3でも動く様にするには？という感じのお話。

MT3向けのプラグインとMT4、MT4向けのプラグインとMT5という関係はおおむね良好なんだけれども、MT3向けとMT5となるとどうも残念な感じらしいです。

MTの魅力はこのプラグインにあるのだから、MT3で使えるようにこころがけてみてはどうでしょう？ということでした。

TagOverrideプラグイン出来ました。の巻

続いてはSix Apartの澤田さんのお話し。既存のMTタグをMTMLで上書きしてしまって挙動を変えてしまおう、ということでした。

澤田さん自身が「このプラグインはどうつかったものか？」とおっしゃっていたのですが、画期的な使い方が表れるかもしれませんね。

MT5とSQLite

つづいてはエムロジックの関根さん（cheebowさん）のお話しで、MT5とSQLiteの関係についてでした。

MT5からSQLiteが正式サポートからは外れてしまったのですが、MTインストールするときにはSQLiteも選択できるよね？という感じでスタート。

関根さんのうまいトークとちりばめられたネタで会場は爆笑の渦。
SQLiteをつかうとカスタムフィールドの挙動が変だったり、ブログを削除できない場合があったりという事もしっかり説明されていました。
ま、MySQLをつかいましょう、というかんじですね。

MT5の日本語化ハマりポイント

続いてはぴろりさんが再度登場。MT5での日本語の扱い方についてのお話しでした。
この辺になってくると完全にわからない世界の話でした。

MT5からUnicodeというかんじでしか理解できませんでした。
将来的にそうだったのか！と思える日が来るのを待ちたいと思います。  

A-FormプラグインのMT5対応版について

続いてはアークウェブの志田さんのお話しで、A-FormプラグインのMT5対応のお話しでした。

A-Form：MovableTypeにUTF-8対応のメールフォームを設置できるMTプラグイン｜Web制作のアークウェブ
http://www.ark-web.jp/movabletype/

A-Formプラグインをつかうとお問い合わせフォームなどがすごく簡単に作成できますよね。
藤本さんのMailFormプラグインとどちらかを使う方が多いのではないでしょうか？

JSONの文字化けなどがあったりで、この辺はぴろりさんが話していた内容とほぼおなじことらしく、解決策も近いもの、みたいなかんじでおっしゃっていました。
あとはMT5からjQueryで色々やるようになった関係でそのあたりの衝突を回避した方法とかを教えていただきました。

本日の最優秀MTエンジニア決定戦！

と、Lightning Talksはすごい内容ばかりでした。
linkerも先日のユーザーギャザリングで軽くお話しさせてもらいましたが、内容の濃さが違いますね。
それはそれ、これはこれ、というかんじですが、なんだかんだいってエンジニアすごい！というかんじでした。

ということで最後に「本日の最優秀MTエンジニア決定戦！」が行われました。

ぴろりさんが作成された珠玉の10問を解いて一番正解が多かった方が（当日に限り）最優秀MTエンジニアと名乗ってしまえ！的なクイズ大会でした。
こんな簡単めの問題から始まり、

こんな感じの深い問題もありました。

そしてそして、10問正解者は残念ながらいなかったのですが、oscarさんとにっくさんが9問で正解数TOPでした！
二人でじゃんけんしてoscarさんの勝利！「本日の最優秀MTエンジニア」 になられました。
おめでとうございます！！

気になる商品は北海道にちなんでこんな感じでした。

oscarさんもスカイアークさんの帯広には行ってみたいとおっしゃっていましたが、シャケ番屋のギフトセットを選ばれていました。
おめでとうございます！！

ということで

先日参加したMT5勉強会もすごくためになりました。
今回のエンジニア向けの話も部分部分は難しすぎて大変ですが、やろうとすることのマップを描くのには良かったと思いました。
これからもこのような機会を開いていただけるのを期待しています！
スカイアークのみなさま、Lightning Talkを行われたみなさま、どうもありがとうございました！
